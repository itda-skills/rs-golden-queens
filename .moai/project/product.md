# product.md — rs-golden-queens

## 프로젝트명

**rs-golden-queens**

## 요약

한국장·미국장 마감 후 텔레그램 채널로 매매동향 요약을 자동 푸시하는 개인 투자 데이터 보조 서비스. 네이버 금융(한국 데이터)과 yfinance(미국 데이터)를 결합하여 매일 세 종류의 리포트를 자동 발송한다.

## 목적과 대상 사용자

| 항목 | 내용 |
|---|---|
| 목적 | 개인 투자자 본인이 직접 사용하는 데이터 보조 도구. 매일 장 마감 후 시장 흐름을 스마트폰 텔레그램 알림 한 건으로 확인. |
| 대상 | 개인 투자자 1인 (자기 계정, 자기 봇, 자기 chat ID 사용). 다수 사용자 배포·SaaS·API 서비스 목적이 아님. |

> 본 서비스는 사실 데이터만 제공합니다. 투자 권유·종목 추천·시점 판단이 아니며, 데이터 정확성·완전성·시의성을 보장하지 않습니다. 투자 결정 전 공식 출처를 반드시 확인하십시오.

## 핵심 기능

| 기능 | 설명 |
|---|---|
| 한국장 일간 발송 | 평일 KST 18:10, 코스피·코스닥 외인·기관·개인 매매동향 + 프로그램매매 + 10거래일 추이 |
| 미국장 일간 발송 | NYSE 마감 +30분, 지수·변동성·섹터·워치ETF·매크로 |
| 주간 리포트 발송 | 그 주 마지막 KR 거래일 KST 18:30, 코스피 5거래일 누적 + 워치ETF 5거래일 등락 |
| DST 자동 반영 | 미국 서머타임 전환 시 dual-cron + 환경변수 게이트로 자동 조정 (SPEC-MF-SCHED-001) |
| 한국 휴장 인지 | XKRX 캘린더 기반으로 비거래일 감지, "[KR] 오늘은 휴장입니다" 한 줄 발송 |
| 미국 휴장 인지 | NYSE 캘린더 기반으로 비거래일 감지, "[US] 오늘은 휴장입니다" 한 줄 발송 |
| 주간 이월 발송 | 금요일 KR 휴장 시 그 주 마지막 거래일로 자동 이월 |
| dry-run 지원 | `MARKET_FLOW_DRY_RUN=1`로 텔레그램 발송 없이 stdout 출력 |

## 핵심 사용 시나리오

### 시나리오 1: cron 자동 발송 (주 사용)

1. GitHub Actions `flow-kr.yml` / `flow-us.yml` / `flow-weekly.yml` 스케줄 트리거
2. `daily_kr.py` / `daily_us.py` / `weekly.py` 실행
3. 거래일 여부 확인 → 휴장이면 한 줄 메시지만 발송
4. `GOLDENQUEENS_BOT_TOKEN` + `GOLDENQUEENS_CHAT_ID`로 텔레그램 발송
5. 사용자는 스마트폰에서 리포트를 수신

### 시나리오 2: 수동 dispatch

GitHub 저장소 Actions 탭 → 원하는 workflow → "Run workflow" 클릭.

### 시나리오 3: 로컬 dry-run

```bash
make daily-kr DRY=1          # 텔레그램 발송 없이 한국장 리포트 stdout 출력
make daily-us DRY=1
make smoke-kr                # 네이버 fetch 단독 점검 (발송 없음)
make smoke-us                # yfinance fetch 단독 점검 (발송 없음)
```

## 정체성

rs-golden-queens는 **스킬 카탈로그의 한 항목이 아닌 독립 자동화 서비스**다. 원래 `itda-skills` 모노리포의 스킬로 구현되었으나, "매일 정해진 시간에 자동 수집 + 텔레그램 알림"은 단독 서비스 형태가 적합하다는 결정으로 분리·이전되었다. 이 저장소가 유일한 보유처이며, `itda-skills` 측에는 코드 흔적이 없다.

초기에는 한국 데이터(네이버)만 수집하는 단순한 구조였으나, commit `8fd2c7f`에서 미국장 연동(yfinance), DST 자동 반영, 휴장 인지 기능을 갖춘 `market_flow` 패키지로 전면 재설계되었다.

## 명시적 비목표

다음은 의도적으로 구현하지 않은 기능이다. 향후에도 추가하지 않는다.

| 비목표 항목 | 이유 |
|---|---|
| **데이터 저장** (commit/Release/Pages) | 본 저장소는 코드 저장 목적만. 수집 결과 이력이 필요하면 별도 데이터 저장소를 분리. |
| **매매 신호·종목 추천·자문** | 사실 데이터만 제공. 투자 권유 없음. |
| **NYSE/KRX 반장 시각 동적 조정** | 반장일은 정상 거래일로 처리. 동적 조정 불필요. |
| **휴장 메시지 다국어화** | 한국어 한 줄("[KR/US] 오늘은 휴장입니다") 고정. 다국어화 불필요. |
| **휴장 사유 표기** | 단순 한 줄 메시지만. 사유(공휴일명 등) 표기 불필요. |
| **시간별 매매동향** | 사용자가 명시적으로 "필요없음"으로 거부. |
| **시계열 누적** | 스냅샷만. 외부 디렉토리 의존 없음. |
